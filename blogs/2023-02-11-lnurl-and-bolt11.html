lnurl and bolt11 libraries

2023-02-11

introduction

around this time i started working more on the small python libraries around LNbits: lnbits/lnurl and lnbits/bolt11.

they are not flashy projects, but they are the kind of libraries that everything else quietly depends on. if invoice parsing, LNURL handling, validation, typing, packaging or release workflows are wrong, the apps built on top get annoying very quickly.

bolt11

with bolt11 the work was mostly about making lightning invoices easier and safer to deal with from python. i worked on the encoder and decoder, route hints, feature bits, fallback addresses, validation, a small cli, tests, pyright, ruff, packaging and release chores.

later fixes were about backwards compatibility, expiry handling, dependency changes, and moving away from older crypto dependencies. boring things, but important things.

lnurl

with lnurl the work was broader. there was tooling cleanup, exports, lnaddress support, lnurlauth, withdraw execution, async requests, timeouts, user agents, pydantic v2, tor fetching, and a lot of version and compatibility work.

i also kept coming back to it whenever LNbits or an extension found a weird edge case. that is the normal life of a protocol library: implement the happy path, then slowly learn all the strange things clients and servers do in the wild.

maintaining

a lot of the work was maintenance. updating dependencies, fixing CI, adding typing checks, keeping releases working, preserving backwards compatibility, and reacting when another project broke because the library was a little too strict or not strict enough.

maintaining small protocol libraries is mostly about not surprising users. old code should keep working, new edge cases should be handled, and releases should be boring. that was the goal.

why it mattered

LNbits has many extensions and integrations. having small reusable libraries for invoices and LNURL meant less duplicated code, better tests, and fewer slightly different implementations spread across projects.

this work also helped later projects like LNURL-pay extensions, hosted LNbits services, and wallet flows where users paste invoices, fetch LNURLs, or expect lightning addresses to just work.
