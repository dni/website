devlog, first half of 2026

2026-06-22

introduction

the first half of 2026 is not fully over yet, but i wanted to write down what i worked on so far. i looked through my public github activity from january until today, and it is mostly a mix of LNbits maintenance, 600 project work, and a few small personal tools.

github commit search shows 256 public commits from my account in this period. the biggest chunks were lnbits/lnbits, 600-party, 600000000000, and vanity-sh.

LNbits

LNbits was still the main open source work. on lnbits/lnbits i worked on releases, CI, dependencies, docker images, payment code, settings, tests, and a bunch of fixes around wallets and invoices.

some things i touched were the v1.5.x release flow, moving more tooling to uv, pyright fixes, docker image changes, onchain/electrum utilities, cached exchange rates, and listener behavior for internal and external invoices.

outside core LNbits i also worked on extensions and related packages: lnurl, bolt11, lnurlp, boltz, events, satspay, and the new lnbits-price-aggregator.

600

the other big chunk was the 600 stuff. i did a lot on 600000000000 and 600-party: pages, styles, party details, nostr files, tor links, business cards, QR code assets, a signal page, and a lot of small content updates.

this was less library work and more website/product/event work. lots of small commits, lots of iteration, and the kind of polish that only happens when something is actually going to be used.

small tools

i also worked on smaller personal projects. vanity-sh got a good amount of commits. geoseed, dnilabs.com, dotfiles, scripts, a fastapi dynamic routes proof of concept, and a timelock wallet also got some attention.

for dnilabs.com i also moved the old blog into the main website, generated the blog overview from blogs.js, and archived the separate blog repository.

reviews and maintenance

the recent activity feed also shows a lot of review and maintenance work, especially around LNbits. in the last public events github returned, there were pull request reviews, merged PRs, reopened PRs, issue comments, branch cleanup, and extension release work.

that part is harder to summarize than commits, but it is important. keeping projects alive is not only writing new code. it is reviewing, releasing, cleaning up, fixing small regressions, and making sure the next person can keep moving.

conclusion

so far 2026 was mostly LNbits, 600, and security-minded personal infrastructure. not a clean theme, but it feels like the right mix: bitcoin tools, web things, events, and making my own setup better.
