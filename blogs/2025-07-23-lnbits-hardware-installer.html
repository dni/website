lnbits hardware installer

2025-07-23

introduction

in 2025 i worked on the LNbits hardware installer. the idea was simple: make flashing and configuring LNbits-related hardware less annoying.

hardware projects need a reliable install path: firmware, device mode, display options, and configuration should be clear before users start flashing.

what i worked on

i simplified the installer flow, added a new boot mode, fixed static deployment, updated the frontend build, and added release builds for multiple devices.

i also fixed device configs, the generic installer name, and firmware uploads.

why it mattered

the installer is the first real user experience for the hardware. if it fails, the project fails before the device is even running.

the goal was a shorter and more reliable path from browser to flashed device.

future ideas

one idea i still like is a dedicated hardware.lnbits.com repository, similar to lnbits/lnbits-extensions, but for hardware projects.

hardware projects could publish releases there, and the frontend could list them, show device-specific options, install firmware, and flash supported devices directly from the browser. one place for all LNbits hardware projects, releases and installers.
